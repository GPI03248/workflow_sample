# Release v0.1

First stable release of the multi-agent CFD workflow sample.

## Highlights

- **2D compressible Euler solver** (NumPy) with Rusanov and HLL fluxes
- **Paper-to-code workflow** with deterministic approval gating
- **HLL Riemann solver** — first method through the full paper-to-code pipeline
- **Validation index** — auto-generated summary of all validation results
- **Repo health check** — automated verification of artifacts and configuration

## Numerical Methods

| Component | Methods |
|-----------|---------|
| Numerical flux | Rusanov (local Lax-Friedrichs), HLL |
| Reconstruction | Piecewise constant, MUSCL (2nd order) |
| Limiters | minmod, van Leer |
| Time integration | Forward Euler, SSP RK2 |
| Boundary conditions | Periodic, transmissive, reflective |

## Validation Cases

- 1D scalar advection (upwind, Lax-Wendroff)
- Uniform flow preservation (machine-precision zeros)
- 2D Sod shock tube
- 2D advected entropy wave (analytic, convergence order ~1)
- 2D isentropic vortex (analytic, convergence order ~2 with MUSCL+RK2)
- HLL vs Rusanov comparison (HLL produces 33% lower L2 error on entropy wave)

## HLL Case Study

The HLL solver is the first method implemented through the full paper-to-code workflow:
- Source note → extraction report → scheme spec → approval gate → implementation → validation
- See `docs/case_studies/hll_flux_paper_to_code.md` for the full walkthrough
- Reproduce: `make demo-hll-workflow`

## Known Limitations

- No turbulence modelling (Euler equations only)
- No adaptive mesh refinement
- No parallelisation (single CPU thread)
- No moving meshes or complex geometries
- HLLC / Roe not yet implemented
